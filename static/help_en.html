User Instructions

Automatic matching of two tables by date and amount

RU EN

1 Prepare and upload your files

The service accepts two files in .xlsx or .xls format. A typical use case is matching a bank statement against a payment register from your accounting system.

What should the file contain? At least one column with dates and one with amounts. All other columns are ignored. The two files can have completely different structures — that is perfectly fine.

How to upload

  1. Drag File A into the left zone, or click it and pick a file from the dialog.
  2. Do the same for File B — the right zone.
  3. If your file has no header row (the first row already contains data), check "First row is data".
  4. Once both files are loaded, the "Next →" button becomes active. Click it.
File won't load? Make sure it is saved as .xlsx or .xls, not password-protected, and not currently open in Excel (close it there first to release the lock).

2 Select columns and set date tolerance

The service shows the column list from each file. Select which column contains the date and which contains the amount for each file. Usually the service guesses correctly — verify and correct if needed.

Date tolerance

Tolerance defines how far apart dates can be for rows to still be considered matching (provided amounts match exactly).

ModeWhen to use
Exact only Both tables contain identical dates — e.g., two statements from the same bank for the same period.
±1 day Dates may differ by 1 calendar day — e.g., a payment was sent on the 31st but the bank credited it on the 1st.
±1 workday (Fri→Mon) Friday and Monday count as 1 working day apart. Useful when a payment left on Friday but the bank processed it on Monday.
±3 days Larger date discrepancies — e.g., reconciling with a slow counterparty or different accounting principles.
±7 days Approximate weekly reconciliation: rows are considered matching if the amount is equal and dates fall within a week of each other.
Tip: start with "Exact only". If many discrepancies remain — increase the tolerance one step at a time and observe which rows move from "Discrepancies" to "Matches".

Click "Run reconciliation". Processing usually takes a few seconds.

3 Read the results and download the report

After reconciliation, a summary panel and a download button appear on screen. The actual result is a colour-coded Excel file.

On-screen summary

Nine cards with key numbers:

  • Total amount A / B — control totals for each file.
  • Total difference — if non-zero, shown in red. This signals that the tables do not balance overall.
  • Exact matches — rows where both date and amount matched perfectly.
  • By date tolerance — amounts matched, but dates differ within the chosen tolerance.
  • Group matches — one row from one file equals the sum of several rows from the other (e.g., a payment split into parts).
  • Not found in B / Not found in A — unmatched rows. These require manual review.

Colour coding in the Excel file

Green Exact match — date and amount matched to the penny.
Yellow Match with date tolerance — amounts are equal but dates differ slightly.
Orange Group match — one row corresponds to the sum of several rows (payment split or consolidated).
Red Discrepancy — no matching row found. Requires manual review.
In the Excel file, matches appear first (sorted by date), followed by the Discrepancies section. Pay extra attention to group matches: the rows of a group are listed one below the other, and grey rows are the "detail" lines of the group.

Click "Download result .xlsx" — the file will be saved to your computer. Click "New reconciliation" to start over.

Frequently asked questions

All rows are red — what's wrong?
The most likely cause is a wrong date or amount column selected in step 2. Click "← Back" and verify your selection. Also check that the date formats in your files are supported (see below).
What date formats are accepted?
The service recognises most common formats:
  • Standard Excel date cells (numeric date format)
  • 01.07.2025 — day.month.year
  • 1.07 or 1.7 — day.month, current year assumed
  • Russian month names: 3 мар, 03 марта, 3 марта 2025
If dates are not recognised — try opening the file in Excel and formatting the date cells as "Date" (not "Text"), then save again.
What is a "group match"?
This occurs when one row in File A equals the sum of several rows in File B (or vice versa). Example: the bank statement shows one payment of 15,000, but accounting has it split into three lines: 5,000 + 7,000 + 3,000. The service finds such combinations automatically (up to 5 rows per group).
Amounts match but rows are red — why?
The date did not match. Try increasing the date tolerance in step 2, or check that the correct date column is selected. If dates are stored in different formats across the two files, verify that the service recognised them correctly.
Total difference is non-zero but no discrepancies are shown
This can happen when some rows were excluded because their dates or amounts could not be parsed. Open the source file, look for empty or non-standard cells, and correct them.
Can I reconcile files without a header row?
Yes. In step 1, check "First row is data" for the relevant file. Columns will be labelled "Column 1", "Column 2", etc. — pick the right ones in step 2.
Is my data safe?
Files are processed on the server solely to perform the reconciliation. Results are stored temporarily and are not accessible to other users via shared links (the download token is random). That said, avoid uploading files containing personal data or trade secrets — use the service at your own discretion.
