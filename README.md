# Сверка документов

Веб-сервис для автоматического сопоставления двух таблиц Excel по дате и сумме. Ориентирован на бухгалтеров и финансовых специалистов.

**Живой сайт:** https://sverka-dokumentov.onrender.com/

---

## Что умеет

- Принимает два файла `.xlsx` / `.xls` с любой структурой столбцов
- Позволяет вручную выбрать колонку даты и колонку суммы для каждого файла
- Поддерживает файлы без строки-заголовка
- Умный автовыбор колонок по ключевым словам (дата, сумма, amount, period и т.д.)
- Пять режимов допуска по дате:
  - Только точные совпадения
  - ±1 календарный день
  - ±1 рабочий день (пятница → понедельник считается как 1 день)
  - ±3 дня
  - ±7 дней
- Групповое совпадение: одна строка = сумма нескольких строк другого файла
- Результат — цветной Excel с разделами «Совпадения» и «Расхождения»
- Файл сохраняется в папку `result_reports/` и скачивается через браузер

## Алгоритм сопоставления

Жадный, четырёхфазный (порядок важен — каждая фаза «забирает» строки до следующей):

1. **Точное совпадение** — одинаковые дата и сумма (погрешность < 0,005)
2. **Группа А→Б** — одна строка А = сумма нескольких строк Б в пределах допуска по дате
3. **Группа Б→А** — одна строка Б = сумма нескольких строк А
4. **Допуск 1:1** — строки с одинаковой суммой и датой в пределах допуска

Группы ищутся раньше допуска, чтобы строки не «уходили» в толерантные пары вместо правильных групп.

Ограничения алгоритма (константы в `reconciler.py`):
- `MAX_GROUP_SIZE = 5` — максимальное число строк в группе
- `MAX_CANDIDATES = 20` — максимальное число кандидатов для подбора группы

## Цветовая легенда результата

| Цвет       | HEX      | Значение                             |
|------------|----------|--------------------------------------|
| Зелёный    | C6EFCE   | Точное совпадение (дата + сумма)     |
| Жёлтый     | FFEB9C   | Совпадение с допуском по дате        |
| Оранжевый  | FFDAA5   | Групповое совпадение (1:N или N:1)   |
| Красный    | FFC7CE   | Расхождение — строка не найдена      |
| Светло-серый | F0F0F0 | Вторичные строки группы (не первая)  |

## Поддерживаемые форматы дат

| Формат входных данных       | Результат              |
|-----------------------------|------------------------|
| Excel-дата (Timestamp)      | Используется как есть  |
| Число `1.07`                | 1 июля текущего года   |
| Строка `"1.07"`             | 1 июля текущего года   |
| Строка `"3 мар"` / `"03 марта 2025"` | Русское название месяца |
| Строка `"01.07.2025"`       | Стандартный разбор     |

Суммы: поддерживаются пробел/неразрывный пробел как разделитель тысяч, запятая как десятичный разделитель.

## API

### `POST /api/preview`
Читает заголовки файлов, возвращает список колонок для выбора.

**Form-data:** `file_a`, `file_b`, `has_header_a`, `has_header_b`

**Ответ:** `{ columns_a, columns_b, name_a, name_b }`

### `POST /api/reconcile`
Запускает сверку, сохраняет результат, возвращает ссылку на скачивание.

**Form-data:** `file_a`, `file_b`, `date_col_a`, `amount_col_a`, `date_col_b`, `amount_col_b`, `date_tolerance`, `has_header_a`, `has_header_b`

**Ответ:** `{ summary, download_url, saved_path }`

**`date_tolerance`:** `"0"` | `"1"` | `"workday"` | `"3"` | `"7"`

### `GET /api/download/{token}`
Скачивает ранее сохранённый файл. Токен валидируется regex-ом и path traversal-проверкой.

## Деплой (Render.com)

**URL:** https://sverka-dokumentov.onrender.com/

**Конфиг (`render.yaml`):**
- Runtime: Python 3.12
- Plan: Free (сервис засыпает после ~15 мин бездействия, первый запрос после сна — медленный)
- Build: `pip install -r requirements.txt`
- Start: `uvicorn main:app --host 0.0.0.0 --port $PORT`

**Особенности free-плана Render:**
- Файлы в `result_reports/` не сохраняются между перезапусками (эфемерная файловая система)
- Если нужно хранить результаты постоянно — потребуется внешнее хранилище (S3, Supabase Storage и т.п.)

## Известные ограничения и потенциальные улучшения

- **Эфемерное хранилище на Render** — результаты теряются при перезапуске; нужен S3/облако для продакшена
- **Один лист Excel** — все результаты в одном листе; можно разбить на «Совпадения» / «Расхождения» по разным листам
- **Нет истории сверок** — каждый запрос независим; можно добавить лог сессий
- **Нет сверки по контрагенту/номеру документа** — только дата + сумма; логичное следующее расширение
- **Производительность** — алгоритм O(n²), при файлах > 5 000 строк заметно замедление; нужна индексация по дате
- **Один пользователь** — нет авторизации; при публичном использовании все видят чужие ссылки для скачивания (токен случайный, но не защищён)
